# taobao-user-behavior-analysis
Taobao user behavior analysis with Python, Pandas, SQL and visualization.  

淘宝用户行为数据分析项目

1. 项目简介

本项目基于淘宝用户行为数据，围绕用户浏览、收藏、加购、购买等行为，完成数据清洗、核心指标分析、行为漏斗分析、时间趋势分析、用户分层、商品转化效率分析、用户-商品行为路径分析以及 RFA 用户价值建模。

项目目标是分析用户从浏览到购买的转化过程，识别不同用户价值群体和关键行为路径，并为电商运营中的用户转化提升、商品优化、用户分层运营和高价值用户维护提供数据依据。

⸻

2. 数据说明

本项目使用淘宝用户行为数据，原始数据字段包括：

字段名	含义
user_id	用户 ID
item_id	商品 ID
category_id	商品类目 ID
behavior_type	用户行为类型
timestamp	行为发生时间戳

其中，behavior_type 包含四类行为：

行为类型	含义
pv	浏览
fav	收藏
cart	加购
buy	购买

本项目将分析时间范围限定为 2017-11-25 至 2017-12-03。清洗后共保留约 1 亿条有效用户行为记录。

注：由于原始数据文件较大，GitHub 仓库中不上传原始 CSV 数据，仅保留代码、分析结果和可视化图表。

6. 用户行为漏斗分析

用户行为漏斗显示，浏览行为占绝大多数。相对于浏览量：

行为	相对浏览量占比
收藏	3.22%
加购	6.17%
购买	2.25%

加购率高于收藏率，说明用户更倾向于通过购物车表达购买意向。从浏览到购买存在明显流失，后续可通过商品推荐优化、详情页优化、购物车提醒和优惠券策略提升转化。

⸻

7. 时间维度行为趋势分析

每日趋势显示，2017-12-02 和 2017-12-03 的浏览、收藏、加购和购买行为均明显上升，说明周末用户活跃度和购买意向更强。

小时维度分析显示，用户行为高峰主要集中在晚间 21:00-22:00。其中浏览和购买高峰出现在 21 点，收藏和加购高峰出现在 22 点。

业务上，可将优惠券推送、限时促销和商品推荐等活动重点安排在晚间高活跃时段。

⸻

8. 用户复购分析

复购分析显示，共有 672,404 名购买用户，其中 443,858 名用户购买次数大于等于 2 次，复购率为 66.01%。

购买用户平均购买约 3 次，中位数为 2 次，说明购买用户整体具备较强复购行为。少数用户购买次数较高，可结合用户价值模型进一步识别高价值用户。

⸻

9. 用户分层分析

本项目基于用户浏览次数、收藏次数、加购次数、购买次数和活跃天数构建用户画像表，并使用分位数阈值和规则分类方法完成用户分层。

用户分层结果如下：

用户类型	用户数	占比
普通购买用户	559,914	56.67%
低活跃用户	229,975	23.28%
高价值用户	112,490	11.39%
潜力转化用户	53,132	5.38%
高浏览未购买用户	32,480	3.29%

高价值用户购买次数和活跃天数均较高，是平台重点维护对象；潜力转化用户加购行为较强但尚未购买，适合通过购物车优惠券、降价提醒等方式促进转化。

⸻

10. 商品转化效率分析

本项目从商品维度构建商品画像表，统计每个商品的浏览、收藏、加购和购买行为，并计算购买转化率、加购率和收藏率。

为避免小样本商品导致转化率虚高，分析中设置最小浏览量门槛，仅保留浏览量达到一定规模的商品进行转化效率分析。

商品分析中重点识别两类商品：

1. 高曝光低转化商品：浏览量较高，但购买转化率较低，适合优化详情页、价格策略、优惠信息和评价展示。
2. 低曝光高转化商品：浏览量不高，但购买转化率较高，适合增加推荐曝光、搜索流量或活动资源。

该模块为商品运营、流量分配和潜力商品挖掘提供了数据依据。

11. 用户-商品行为路径分析

为避免用户维度聚合导致不同行为商品之间的路径混淆，本项目以 user_id + item_id 为分析粒度，构建用户-商品行为路径表。

路径分析结果如下：

行为路径	用户-商品组合数	占比
仅浏览	66,327,040	87.35%
浏览-加购未购买	4,934,810	6.50%
浏览-收藏未购买	2,636,128	3.47%
浏览-购买	1,464,955	1.93%
浏览-加购-购买	331,818	0.44%
浏览-收藏-购买	113,893	0.15%
浏览-收藏-加购未购买	104,595	0.14%
浏览-收藏-加购-购买	15,361	0.02%

结果显示，大部分用户-商品交互停留在仅浏览阶段。加购未购买路径占比高于收藏未购买路径，说明加购行为是更强的购买意向信号。

12. RFA 用户价值模型

由于数据中不包含消费金额字段，本项目基于 RFM 思想构建简化版 RFA 用户价值模型：

指标	含义
R	最近一次购买距离分析基准日的天数
F	用户购买频率
A	用户活跃天数

通过分位数分箱将 R、F、A 转换为 1-4 分，并基于得分规则划分用户价值类型。

RFA 用户价值分层结果如下：

用户价值类型	用户数	占比
一般价值用户	207,296	30.83%
高频沉睡用户	129,924	19.32%
近期购买低频用户	129,924	19.32%
高价值用户	112,194	16.69%
潜力价值用户	93,066	13.84%

高价值用户适合重点维护；高频沉睡用户适合召回运营；近期购买低频用户适合通过二次购买优惠和关联商品推荐提升复购。

⸻

13. 主要业务建议

基于以上分析，提出以下运营建议：

1. 在晚间 20:00-22:00 用户高活跃时段加大优惠券、限时促销和商品推荐力度。
2. 对加购未购买用户推送购物车优惠券、降价提醒和库存提醒，提高购买转化。
3. 对高曝光低转化商品优化详情页、价格策略、评价展示和主图标题匹配度。
4. 对低曝光高转化商品增加推荐流量和活动曝光，验证其放大潜力。
5. 对高价值用户提供会员权益和专属优惠，提高用户粘性。
6. 对高频沉睡用户进行召回运营，降低高价值用户流失风险。
7. 对近期购买低频用户进行二次购买引导，提高复购率。
